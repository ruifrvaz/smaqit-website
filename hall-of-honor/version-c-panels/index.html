smaQit Power up with smaQit

Open source · IT professionals · Product Owners, Architects, Engineers, Testers

Three toolkits. One methodology.

Each toolkit fits a different shape of project. smaQit drives greenfield builds with heavy business requirements; Extensions adds tooling to that project — or stands alone for app-heavy, logic-light builds; ADK is the complement that lets you build your own agentic ecosystem. Use the tabs above to explore each one.

smaQit

Spec-driven agent orchestration for greenfield projects with heavy business requirements — describe requirements, get working, tested, deployed code.

  • Best fit for new builds carrying real business logic
  • Stateful specs track draft → implemented → deployed → validated
  • Bounded agents — one agent, one layer, no scope creep
  • Five layers, three phases, mapped to Agile roles

smaQit Extensions

Adds tooling to an existing smaQit project — or runs standalone for application-heavy builds with little to no business logic.

  • Pairs with smaQit to add tooling to a business-heavy project
  • Or runs standalone for app-heavy, logic-light builds
  • Session continuity, task tracking, release automation
  • Works standalone in any git repository
